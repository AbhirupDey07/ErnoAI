Interactive Tutorial — Basics

How many pieces?

  • 6 centers — fixed orientation, define the face colors.
  • 12 edges — each has 2 stickers.
  • 8 corners — each has 3 stickers.

Centers don’t move relative to each other — they tell you which color belongs on each face.

Standard color layout

Common scheme: White opposite Yellow, Green opposite Blue, Red opposite Orange. Brand models may vary slightly; centers define the final solved orientation. More specifically, on a standard 3x3 Rubik's Cube typically goes like this (going clockwise):
White bottom, Yellow Top, Blue, Orange, Green, Red

Notation — complete reference (3×3)

Clockwise (R) and counter-clockwise (R') are defined from looking straight at that face. 2 means 180° (same both directions).
You can experiment with this yourself by using the Move Picker keyboard on the simulated cube in the following steps.

Face turns (outer layer)
Faces:
  • R R' R2 — Right
  • L L' L2 — Left
  • U U' U2 — Up
  • D D' D2 — Down
  • F F' F2 — Front
  • B B' B2 — Back
Meaning:
  • X — 90° clockwise
  • X' — 90° counter-clockwise
  • X2 — 180°
Slice turns (middle layers)
Slices:
  • M M' M2 — between L/R, turns like L
  • E E' E2 — between U/D, turns like D
  • S S' S2 — between F/B, turns like F
Notes:
  • M follows L direction (opposite R).
  • E follows D direction (opposite U).
  • S follows F direction (opposite B).
Wide turns (two-layer)

The simulator doesn’t have separate wide-move inputs. To emulate a wide turn, perform the listed outer face + slice in sequence.

Notation (either form):
  • r r' r2 or Rw Rw' Rw2
  • l l' l2 or Lw Lw' Lw2
  • u u' u2 or Uw Uw' Uw2
  • d d' d2 or Dw Dw' Dw2
  • f f' f2 or Fw Fw' Fw2
  • b b' b2 or Bw Bw' Bw2
Emulation on 3×3 (outer + slice):
  • r = R + M' (two rightmost layers)
  • l = L + M
  • u = U + E'
  • d = D + E
  • f = F + S
  • b = B + S'

Add ' or 2 to apply the same relation (e.g., r' = R' + M).

Cube rotations (whole-cube)
  • x x' x2 — rotate cube like R (left-right axis)
  • y y' y2 — rotate cube like U (up-down axis)
  • z z' z2 — rotate cube like F (front-back axis)

Rotations change your perspective. Algorithms often assume an orientation (e.g., y before a sequence). Note that the simulator will NOT adjust face notations for rotations (for now :3)

Notation quick list (all primitives)

Every move below also has ' and 2 variants.

R L U D F B · M E S · r l u d f b / Rw Lw Uw Dw Fw Bw · x y z

Reading algorithms

  • Concatenation: moves go left → right, e.g., R U R' U'.
  • Parentheses / powers: (R U R' U')×3 means repeat 3 times.
  • Setup moves: a short prefix (e.g., y) reorients before the main sequence.
  • Fingertricks: primes vs non-primes usually use different fingers.
